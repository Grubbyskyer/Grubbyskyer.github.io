约瑟夫问题

杂题

约瑟夫问题大家都很熟悉了：有n个人，编号为1~n，站成一个环，开始报数，报到m的人退出，从下一个人继续从1开始报数，直到只剩下一个人，问这个人的编号。

当初最开始学链表时候，还拿这个练过链表，模拟了每一次的报数，复杂度当然是$O(nm)$，我们需要一点数学方法，这里给两个吧，一个$O(n)$的，一个$O(logn)$的。

为了讨论方便，先把问题稍微改变一下，并不影响原意：

问题描述：n个人（编号0~(n-1))，从0开始报数，报到(m-1)的退出，剩下的人继续从0开始报数。求胜利者的编号。

我们知道第一个人(编号一定是m%n-1) 出列之后，剩下的n-1个人组成了一个新的约瑟夫环（以编号为k=m%n的人开始）:
k k+1 k+2 … n-2, n-1, 0, 1, 2, … k-2
并且从k开始报0。

现在我们把他们的编号做一下转换：
k —> 0
k+1 —> 1
k+2 —> 2
…
…
k-2 —> n-2
k-1 —> n-1

变换后就完完全全成为了(n-1)个人报数的子问题，假如我们知道这个子问题的解：例如x是最终的胜利者，那么根据上面这个表把这个x变回去不刚好就是n个人情况的解吗？！！变回去的公式很简单，相信大家都可以推出来：x’=(x+k)%n
比如从n-2变回k-2，(n-2+k)%n = k-2，这里的n是当前还有多少个人在报数，不是最开始总人数那个n

如何知道(n-1)个人报数的问题的解？对，只要知道(n-2)个人的解就行了。(n-2)个人的解呢？当然是先求(n-3)的情况 —— 这显然就是一个倒推问题！好了，思路出来了，下面写递推公式：

令f[i]表示i个人玩游戏报m退出最后胜利者的编号，最后的结果自然是f[n]

递推公式
f[1]=0;
f[i]=(f[i-1]+m)%i; (i>1)

有了这个公式，我们要做的就是从1-n顺序算出f[i]的数值，最后结果是f[n]。因为实际生活中编号总是从1开始，我们输出f[n]+1

由于是逐级递推，不需要保存每个f[i]，程序也是异常简单：

还有一种推公式的$O(logn)$的解法：
$$
J(1) = 1 \\
J(2n) = 2J(n) - 1 \\
J(2n+1) = 2J(n) + 1
$$
这样我们只需找到一个2的k次幂记为T，使得T > n，注意这里必须要大于，不能等于，那么最终剩的人的编号为：2n - T + 1
n = 1, T = 2, J(1) = 2 * 1 - 2 + 1 = 1
n = 2, T = 4, J(2) = 2 * 2 - 4 + 1 = 1
n = 3, T = 4, J(3) = 2 * 3 - 4 + 1 = 3
n = 4, T = 8, J(4) = 2 * 4 - 8 + 1 = 1
这样就可以在$O(logn)$时间内找到答案。

最后再给一道题，设J为约瑟夫函数，问对一个给定的n，使用约瑟夫函数，直到它得到了一个固定的数，固定的数的意思就是J(k) = k，问给一个n，最终得到几
比如J(1) = 1，1 == 1，结束，这样结果就是1
J(2) = 1, 2 != 1, 继续，J(1) = 1, 1 == 1结束
http://acm.csu.edu.cn/OnlineJudge/problem.php?id=1125
